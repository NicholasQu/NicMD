# NicMD Logo 设计回溯

## 核心符号

NicMD 的图形核心不是一个普通的 `M`，而是数学化的 `𝕸ⁿ(𝒊𝒄)`。

这个表达可以读成：

- `𝕸`：Markdown / Mark / Math / Mind 的主符号
- `ⁿ`：来自 Nic 的 n，也有指数、增长、扩展能力的含义
- `(𝒊𝒄)`：把 nic 收束成类似函数参数或数学注释的小尾巴
- `nicmd`：落到 CLI 时就是一个干净的命令名，也可以被读成 `ni + cmd`

最终它看起来像一个公式，也像一个终端里偶然打出来的漂亮 Unicode 标记。

## 为什么不是普通字母 Logo

NicMD 本身是 Markdown 编辑器，气质应该介于四件事之间：

1. 写作工具：需要克制、安静、可读
2. 开发者工具：需要有一点终端、代码、公式感
3. CLI 工具：命令行里可以直接写成 `nicmd`，短、顺、像一个真正可敲的命令
4. 个人品牌：需要有一个不像通用模板的记忆点

`NicMD` 第一层可以读成 `Nic + MD`，直接指向 Nic 的 Markdown 编辑器；第二层也可以断成 `Ni + cmd`，自然连接到 command line / cmd 的语义。这一点很巧：它既像一个桌面写作工具的名字，也像一个可以在终端里输入的命令。

普通 `N`、`M` 或纯图形图标很容易显得像模板应用；而 `𝕸ⁿ(𝒊𝒄)` 具有更强的个人标记感，像一个只属于 NicMD 的签名。

## Unicode 花体的使用原则

花体 Unicode 很有识别度，但不能滥用。

适合使用的位置：

- App icon
- 顶部品牌字标
- 工具栏中心标题
- 欢迎页或品牌展示页
- 文档里的品牌公式

不适合使用的位置：

- 正文编辑区
- 文件树
- 按钮文本
- 长段说明
- 需要复制、搜索、匹配的业务文本

因此 UI 里采用“少量点缀”的方式：功能文本仍然保持系统字体，只有 `NicMD` 品牌区域带有数学花体气质。

## 视觉元素

### 字形

- 主字形：`𝕸`
- 公式字标：`𝕸ⁿ(𝒊𝒄)`
- 备用品牌名：`NicMD`

`𝕸` 使用双线体，有一点数学黑板字母的味道，适合表达 Markdown、公式、结构化写作。

`𝒊𝒄` 使用 script 风格，像手写批注，让品牌从纯工具感里带一点个性。

### 配色

主色来自暖色渐变：

- 金黄：`#fbbf24`
- 橙色：`#ea580c`
- 红色点缀：`#dc2626`
- 浅底：`#fef2f2`

这个配色避免纯黑白工具的冷感，让编辑器有一点“纸张、灯光、标注”的温度。

### 形状

当前图标采用圆角矩形容器：

- 适合 Windows 桌面图标
- 适合 Electron 应用的现代工具感
- 比纯圆形更像一个“文档卡片”或“编辑器窗口”

## 与界面的关系

App 内部不直接把整个界面做成花体风格，而是让 Logo 成为视觉锚点：

- 左上角：小 `𝕸` 徽标 + `NicMD` 字标
- 工具栏中间：`𝕸ⁿ(𝒊𝒄)` + 当前文件名
- Markdown 欢迎文案：保留公式式品牌表达

这种做法让应用有记忆点，同时不破坏编辑器应有的清爽和可读性。
